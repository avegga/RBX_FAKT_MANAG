Python-приложения
1.	Цель
Разработать приложение, для работы с табличными фактами.
Описание:
Приложение должно обеспечивать выбор шаблона, выбор и загрузку исходного файла, отображение данных в табличной области, визуализацию полученных данных в виде различных диаграмм, графиков управление параметрами отображения и дальнейшую обработку данных через правую панель.
Визуализация диаграмм и графиков относится к полной версии продукта; в рамках MVP вкладка “Анализ” реализуется как заглушка.

2.	Состав приложения
Приложение содержит основные вкладки:
•	Загрузка фактов
•	Анализ
•	Инструкции
•	Журнал
•	Настройки
•	Резерв

3.	Общие требования к интерфейсу
Интерфейс с минимальными внешними и внутренними отступами между панелями, между панелями и границами экрана.
Для всех закладок вверху располагается панель («Главная панель»): слева направо: надпись "Система управления фактами», закладки: "Загрузка фактов", "Анализ", "Инструкции", "Журнал", "Настройки", «Резерв".
4.	Технические требования
Приложение должно быть реализовано на языке Python.
Приложение должно представлять собой web-приложение, доступное через браузер.
В рамках MVP приложение должно работать в однопользовательском режиме.
Приложение должно запускаться локально на одном персональном компьютере.
Приложение должно обеспечивать работу с файлами объемом до нескольких десятков тысяч строк.
Табличное представление данных должно поддерживать виртуализацию строк.
Операции загрузки, отображения, фильтрации и изменения параметров отображения должны выполняться без заметных для пользователя подвисаний интерфейса.
Интерфейс приложения должен обновляться без полной перезагрузки страницы при выполнении пользовательских действий.
Приложение должно обеспечивать сохранение пользовательских и системных настроек между сессиями в соответствии с разделом «Состояние и хранение».
Приложение должно обеспечивать сохранение журналируемых событий в соответствии с требованиями раздела «Журнал».
Приложение должно обеспечивать загрузку инструкций из внешнего текстового файла в формате plain text.
Приложение должно обеспечивать возможность дальнейшего расширения до многопользовательского режима без полного пересмотра модели хранения данных.
4.1. Рекомендуемые технические решения
Backend рекомендуется реализовать на Python.
Для хранения шаблонов, настроек, состояний и журнала в MVP рекомендуется использовать SQLite.
Загруженные данные рекомендуется обрабатывать в памяти в рамках текущей пользовательской сессии.
Интерфейс рекомендуется реализовать как браузерный клиент с поддержкой виртуализированной таблицы.
Архитектура должна предусматривать разделение слоя хранения, слоя обработки данных и слоя пользовательского интерфейса.


5.	Описание закладок
5.1.	Закладка "Загрузка фактов"
В самом верху («Главная панель»).
Ниже «Главная панель»: размещается строка служебных сообщений "Строка служебных сообщений".
Ниже чем "Строка служебных сообщений" должна располагаться горизонтальная панель назовем ее "Панель элементов" на всю доступную ширину.
Ниже панели ("Панель элементов") размещена рабочая область разделена на две части вертикальной линейкой:
левая область — "Таблица данных";
правая область — "Настройка данных".
левая область "Таблица данных" разделяется на две области нижняя и верхняя горизонтальной линейкой.  Линейка может изменять размеры областей. Верхняя область "Данные", нижняя область "Ошибки".
Нижнюю область надо скрывать/разворачивать каким-либо образом, например, выбором  чек бокса.

Разберем "Загрузка фактов" по панелям/областям:
5.1.1.1.	Панель: "Строка служебных сообщений":
В высоту не больше 20 пискселей, чем меньше тем лучше  в ширину примерно 50% от ширины рабочей области. 
Предназначена для вывода системных и пользовательских сообщений.
В строке должны отображаться статусы операций, предупреждения и результаты действий.
Сообщение должно обновляться без перезагрузки экрана. 
Примеры сообщений:
Файл не выбран
Выбран файл: имя файла
Данные успешно загружены
Таблица очищена
Панель данных скрыта
Это одна строка с последним сообщением а не несколько последних сообщений.
Сообщения должны оставаться до следующего действия.
Нужны уровни сообщений: информация, предупреждение, ошибка, успех.
5.1.1.2.	Панель: «Панель элементов"
Элементы панели располагаются слева направо в следующем порядке:
а) Элемент список выбора Шаблон: 
Название: «Шаблон».
Элемент: список выбора.
Состав списка по умолчанию: «Шаблон 1», «Шаблон 2».
Значение по умолчанию: «Шаблон 1».

Шаблон определяет структуру и правила обработки данных и хранится в SQLite.
Пользователь может создавать, переименовывать, удалять шаблоны и сохранять новый шаблон с произвольным именем.

Шаблон должен хранить:
имя шаблона;
версию шаблона;
предопределённый список ожидаемых столбцов;
назначенные типы данных для столбцов.
Привязка типа данных должна выполняться на уровне шаблона и названия столбца.

В рамках MVP валидация шаблона ограничивается проверкой по типу данных. Проверки обязательности, диапазонов, регулярных выражений и справочников допустимых значений не требуются.

Если в загружаемом файле отсутствуют столбцы, ожидаемые шаблоном, приложение должно продолжать работу с теми столбцами, которые присутствуют. Такая ситуация считается частичной загрузкой и должна сопровождаться предупреждением в строке сообщений и записью в журнал.

Если в загружаемом файле присутствуют новые столбцы, отсутствующие в шаблоне, они должны по умолчанию отображаться в таблице и трактоваться как текстовые.

Поддержка optimistic locking резервируется для будущей многопользовательской версии.
б) Информационная строка исходного файла
Элемент: текст.
Отображает имя текущего выбранного файла после выбора его в диалоге загрузки.
До выбора файла может содержать текст: “Файл не выбран”.
После перезапуска приложения, если ранее был сохранен последний успешно загруженный файл и он существует, в строке может отображаться его имя до момента выбора нового файла.
в) Элемент: кнопка "Загрузить"
Название: "Загрузить"
 Что делает: Загружает выбранный файл в рабочую область "Таблица данных". Открывается диалог выбора файла в папке источника, указанной в настройках, в котором пользователь выбирает файл
При загрузке:
При нажатии «Загрузить» сначала открывается диалог выбора файла, а потом файл сразу загружается – должно быть так. Файл выбирается из папки источника, указанной в настройках.
Загружает только файлы: xls, CSV,  xlsx – если файлы другого типа то выдает ошибку.
CSV - кодировка: UTF-8 с BOM (обязательно, иначе Excel покажет иероглифы).
Разделитель: ; (точка с запятой), так как в русской локали запятая — это десятичный разделитель.

XLS/XLSX 
Локаль: ru_RU (для дат, чисел, десятичных разделителей).
Для XLS/XLSX нужно читать именно вычисленные значения формул.
Если в Excel-файле несколько листов, брать всегда первый лист.
Пустые строки в начале файла до заголовка – игнорировать.
Всегда считать, что первая строка это заголовок, не нужен флаг «первая строка = заголовки».
С дублирующимися именами столбцов в заголовке – добавлять циферку после названия.
Правила именования дублей столбцов: например Сумма, Сумма_2, Сумма_3.
заголовок тоже отображает в "Таблица данных"
каждый столбец: заголовок попадает в «Столбцы»
При отсутствии части ожидаемых шаблоном столбцов применяется режим частичной загрузки согласно правилам шаблона.
г) Элемент: кнопка "Сохранить" 
Название: "Сохранить"
Что делает: Сохраняет результат данные из рабочей области "Таблица данных", так как они видны пользователю, но с исключениями, ниже смотри для форматов: данных "Денежный». 
«Видно пользователю»  при нажатие кнопка "Сохранить"  означает: не учитываются только скрытые колонки инструментом «Столбцы», учитывается текущий порядок, учитываются форматированные значения после преобразования по типу.
Если формат данных "Денежный»:
  то преобразовывает в число;
  при этом если на экране отображается без копеек, то сохраняет только целую часть числа.
Сохраняет в файле формата xlsx.
Название сохраненного файла состоит: слово data, нижнее подчеркивание, дата, месяц, год (последние цифры), час, минута (эти данные их дата/время при нажатии кнопки «Сохранить»).
Сохраняются только видимые столбцы, в текущем порядке, и только строки после фильтрации. Значения сохраняются в отображаемом после преобразования виде, кроме отдельно оговоренных правил для денежного типа. 
Итоговый XLSX всегда сохраняется в папку результата, указанную в настройках. Пользователь не выбирает путь сохранения самостоятельно.
Формат имени файла окончательно такой: data_DDMMYY_HHMM.xlsx
Если файл с таким именем уже существует, нужно добавить суффикс.
При сохранении, если папка результата стала недоступна или файл невозможно записать, то
 не сохранять, показать ошибку в строке сообщений и записать событие в журнал.
д) Элемент: кнопка "Показать/Скрыть панель"
Что делает: кнопка скрывает или отображает правую панель «Настройка данных".

e) Элемент: чек бокс «Отобразить все»
Что делает: чекбокс управляет только объемом отображения строк в интерфейсе. Файл после загрузки полностью считывается в рабочий набор текущей сессии. Если чекбокс установлен, отображаются все строки рабочего набора. Если чекбокс не установлен, отображение ограничивается значением поля «Кол-во строк». Состояние чекбокса не сохраняется между сессиями.
Под сессией понимается период работы приложения от запуска до закрытия пользователем.

5.1.1.3.	Панели "Таблица данных" и "Настройка данных" 
разделяются вертикальной линейкой, которая позволяет изменять размеры рабочих областей по ширине.
Требования к изменению размеров: приложение запоминает, в автоматическом режиме, размер панелей при разделение линейкой и при следующей сессии именно так как были до закрытия отображает панели.
См. правила сохранения состояния в разделе 6.
5.1.1.3.1.	Левая панель «Таблица данных»:
 В верхняя область "Данные" отображает загруженный файл (при нажатии кнопки «Загрузить») с учетом проставленных фильтров в панели "Настройка данных" закладка «Столбцы», «Общие», «Фильры», «Типы», в нижней области - «Ошибки» отображает ошибки области «Данные».
Область имеет горизонтальный и вертикальный скрол бар;
Есть возможность менять ширину столбцов прямо в интерфейсе.
Сортировка таблицы выполняется по клику на заголовок, влияет на текущее отображение и на порядок строк при сохранении результата, но не сохраняется между сессиями.
Запрещено редактирование ячеек пользователем вручную в таблице.
Нижняя область «Ошибки» скрывается по чекбоксу.
Размер у области «Ошибки» должен быть по умолчанию 0,3 от "Данные".
Сохранять состояние  области «Ошибки» открыта/скрыта между сессиями обязательно.
Визуально помечать ошибочную ячейку в верхняя область "Данные": цвет фона.
Если ошибок нет, область «Ошибки» должна быть просто пустой.
При изменении типа столбца ошибки должны пересчитываться сразу по всем строкам файла, а не только по видимым.
5.1.1.3.2.	Правая панель "Настройка данных"
В верхней части панели «Настройка данных": размещаются еще закладки: «Столбцы», «Общие», «Фильтры», «Типы», «Обработка».
Панель имеет вертикальный скрол бар;
Настройки на закладках всегда применяются мгновенно.
при уменьшении ширины правой панели элементы не должны наезжать друг на друга;
текст на кнопках правой панели "Настройка данных" должен обрезаться границами кнопки;
при наведении курсора на кнопку должен отображаться полный текст кнопки.
Закладки панели «Настройка данных»:
«Общие»
«Столбцы»
«Фильтры»
 «Типы»
 «Обработка»

Закладка «Общие»
Чек бокс «скрыть копейки»: скрывает копейки в панели «Таблица данных» - только для столбцов, которым присвоен тип «Денежный».
Помимо «скрыть копейки» нужен еще отдельный чекбокс «только целые» для числовых столбцов. Это независимая настройка от «скрыть копейки». Если столбец типа «Числовой» и включен «только целые», необходимо отбрасывать дробную часть.
Строка ввода: «Кол-во строк» ограничивает количество загруженных из файла строк при нажатии кнопки "Загрузить" из панели «Панель элементов".
Кол-во строк ограничивает количество отображаемых строк при первичном показе данных в интерфейсе.

Закладка «Столбцы»
Состоит из списка столбцов, слева от названия чек бокс. Если чек бокс выбран, то столбец отображается.
Также можно поменять порядок следования столбцов в панели «Таблица данных». Для смены порядка столбцов нужны кнопки вверх/вниз.
Столбцы размещаются вертикально.
Закладка «Фильтры»
Состоит из списка столбцов, отображается название слева и каждому столбцу окно ввода, которое соответствует типу для выбранного столбца.
Если столбцу не присвоен тип, то тогда текстовое поле по типу -  содержит текст. 
Должны поддерживаться следующие типы данных: текст, число, денежный, дата, дата-время, логический- этого достаточно для MVP.
Тип дата:
Для столбцов с типом дата – выбор даты и выбор даты по, время не входит в фильтр.
правила обработки пустых значений:  обрабатывать как 01.01.1900, и  отображать как ошибку в «Ошибки».
Для типа “дата-время” фильтр не должен учитывать, только день. 
Тип Число:
Для чисел нужны операции: равно, больше, меньше, диапазон, как два поля “от/до”.
правила обработки пустых значений: обрабатывать как 0 и отображать как ошибку в «Ошибки»
Тип Текст: 
Для текста фильтр это только «содержит».
Тип Логический:
Для логического типа нужен список Да/Нет/Пусто.
правила обработки пустых значений: обрабатывать как Нет, и отображать как ошибку в «Ошибки».
Фильтры должны работать по всем строкам файла.
Тип Денежный:
правила обработки пустых значений также как тип Число
Закладка «Типы»
Состоит из списка столбцов, отображается название слева и каждому столбцу список выбора типа данных (Стандартный вариант), например: Числовой, Дата, Логический и т.д.. При выборе Типа для столбца, столбцу присваивается тип данных. При этом если есть ошибка, то появляется подробное сообщение для каждого значения в нижняя область "Ошибки" область "Таблица данных".
Если в текущем шаблоне для столбца с таким названием уже задан тип данных, он должен подставляться автоматически при следующей загрузке файла по этому шаблону.
 «Логический» трактовать: только true/false, Да/Нет, 1/0, Истина/Ложь
При ошибке приведения типа нужно:
строку оставлять в таблице и только писать ошибку, и помечать ячейку, и делать запись об ошибке в области «Ошибки» не исключать строку из «Данные».
Ошибки типизации должны показываться для каждого значения с указанием строки, столбца, исходного значения и причины.
Если пользователь меняет тип столбца, эти настройки должны сохраняться для шаблона сразу автоматически.

«Денежный» 
Корректными считаются целые и дробные значения в следующих форматах: 1234, 1 234, 1234.56, 1 234.56. В рамках MVP для типа «Денежный» в качестве десятичного разделителя используется точка.
«Числовой»
Корректными считаются целые и дробные значения в следующих форматах: 1234, 1 234, 1234.56, 1 234.56. В рамках MVP для типа «Числовой» в качестве десятичного разделителя используется точка.

Локаль ru_RU применяется к обработке Excel-файлов и дат. Для типов «Числовой» и «Денежный» в рамках MVP используется точка как десятичный разделитель.
 
«Дата»
 Корректными считаются значения, содержащие только дату. Допускаются форматы ДД.ММ.ГГГГ, ГГГГ-ММ-ДД.

«Дата-время»
Корректными считаются значения, содержащие дату и время. Допускаются форматы ДД.ММ.ГГГГ ЧЧ:ММ, ГГГГ-ММ-ДД ЧЧ:ММ, ГГГГ-ММ-ДДTHH:MM:SS

Закладка «Обработка»
Содержит:
 радиокнопки «парсинг», «прочее»;
При выборе радиокнопка «парсинг» появляется:
 список выбора (название «столбец парсинга»);
 кнопка «выполнить парсинг»,
 окно ввода текста «Разделитель парсинга».
Список выбора «столбец парсинга») содержатся столбцы «Таблица данных», из только тех, что видны на экране, для парсинга ограничение именно по столбцам «Видно пользователю» и всем строкам после фильтрации, а не по всем данным. 
Пользователь может выбрать один столбец в «столбец парсинга».
При нажатии на кнопку «выполнить парсинг»: анализируются по всем строкам после фильтрации, независимо от лимита, для каждой строки если в столбце который выбран в «столбец парсинга», есть разделитель указанный в «Разделитель парсинга». 
 При выполнении парсинга исходная строка удаляется из текущего рабочего набора и заменяется набором новых строк.
Количество создаваемых строк определяется количеством непустых значений, полученных после разделения содержимого выбранной ячейки по указанному разделителю.
Пустые значения, возникающие между подряд идущими разделителями, не создают новые строки.
Во всех созданных строках значения остальных столбцов должны совпадать со значениями исходной строки.
При этом если в результате парсинга получается пустота, то строка не создается. После парсинга пробелы слева и справа удаляются.   
Внутри панели никакие кнопки «Применить/Сохранить» не нужны - должно обновляться сразу без подтверждения.
Состояние скрыта/показана правая панель «Настройка данных» нужно сохранять между сессиями.
После парсинга исходные строки удаляются только из текущей сессии.
Если в одной ячейке разделитель встречается подряд и между ними пустое значение, такую пустую часть нужно пропускать полностью.
Нужен  предпросмотр результата парсинга до выполнения.
Нужна кнопка отката последнего парсинга.
Парсинг применяется ко всем строкам после фильтрации, независимо от лимита отображения. Для выбора доступны только столбцы, видимые пользователю. 
Опция «прочее» пока пустая заглушка.
Предпросмотр парсинга должен показывать:
количество строк, которые будут удалены,
количество строк, которые будут созданы.
Откат парсинга нужен только на один шаг назад.

5.2.	Закладка «Резерв"
   - заглушка.
5.3.	Закладка «Настройки"

Во вкладке «Настройки» пользователь задает и изменяет:
путь к папке источника файлов;
путь к папке результата для выгрузки;
путь к внешнему текстовому файлу инструкций.
Указанные пути являются путями к папкам и файлу на диске, а не параметрами подключения к базе данных.
Приложение должно валидировать существование каждого указанного пути сразу при вводе или изменении значения.

Защита вкладки «Настройки» паролем в рамках MVP не требуется.
5.4.	Закладка «Журнал"  
Во вкладке «Журнал» отображаются системные изменения приложения и пользовательские действия. Журнал является глобальным. Каждая запись содержит timestamp, user, template, action, details. В рамках MVP сохраняются только последние 50 записей журнала. Записи автоматически добавляются при загрузке, сохранении, изменении шаблона и парсинге. В качестве идентификатора пользователя используется имя Windows-пользователя. В интерфейсе доступна фильтрация по пользователю и шаблону.

5.5.	Закладка «Инструкции»
Вкладка “Инструкции” должна отображать содержимое текстового файла, путь к которому задан во вкладке “Настройки”.
Редактирование пути к файлу во вкладке “Инструкции” не выполняется.
Формат файла инструкций: plain text.
Файл инструкций является общим для всех шаблонов.

5.6.	Закладка “Анализ” – пока заглушка.


6.	Состояние и хранение

Несмотря на однопользовательский режим MVP, модель хранения должна быть спроектирована с разделением общих и пользовательских данных для обеспечения последующего перехода к многопользовательскому режиму.

Данные хранения должны быть разделены на:

общие данные приложения: шаблоны, версии шаблонов, ожидаемые столбцы, типы данных, журнал;
пользовательские настройки: последний выбранный шаблон, последний успешно загруженный файл, порядок столбцов, ширины столбцов, фильтры, настройки вкладки «Общие», параметры обработки, размеры панелей, выбранные вкладки;
временные сессионные данные: значения и состояния, действующие только до завершения текущей сессии.
Приложение должно сохранять состояние между перезапусками.

Между сессиями должны сохраняться:

последний выбранный шаблон;
размеры основных панелей интерфейса;
состояние правой панели «Настройка данных»: показана или скрыта;
состояние нижней области «Ошибки»: показана или скрыта;
последняя выбранная основная вкладка приложения;
последняя выбранная внутренняя вкладка правой панели «Настройка данных»: «Общие», «Столбцы», «Фильтры», «Типы» или «Обработка»;
порядок столбцов в панели «Таблица данных»;
ширины столбцов в панели «Таблица данных»;
назначенные типы данных столбцов;
заданные фильтры;
настройки из вкладки «Общие»;
параметры обработки;
последний успешно загруженный файл.
Под сессией понимается период работы приложения от запуска до закрытия пользователем.

Между сессиями не сохраняются:
загруженные данные текущей сессии;
результаты парсинга и другие изменения рабочего набора;
состояние чекбокса «Отобразить все»;
временные результаты предпросмотра парсинга;
данные отката парсинга;
текущее состояние сортировки таблицы.

При каждом новом запуске приложения чекбокс «Отобразить все» должен быть сброшен в состояние по умолчанию.

Последний успешно загруженный файл после перезапуска приложения должен обрабатываться следующим образом:

если файл существует, приложение должно отображать его имя и предлагать пользователю повторную загрузку;
автоматическая загрузка файла при запуске приложения не выполняется;
если файл не найден, имя файла не отображается, и предложение повторной загрузки не показывается.
В рамках MVP хранение состояния, настроек и служебных данных должно выполняться в SQLite.



7.	Приоритеты MVP
В рамках MVP должны быть реализованы следующие ограничения и приоритеты:
приложение должно быть реализовано как локальное web-приложение на Python;
приложение должно запускаться на одном персональном компьютере и открываться в браузере;
в рамках MVP приложение должно работать в однопользовательском режиме;
механизмы аутентификации и авторизации в рамках MVP не реализуются;
для хранения шаблонов, настроек, состояний интерфейса и журнала должна использоваться SQLite;
в рамках MVP должна быть полностью реализована вкладка "Загрузка фактов", а также минимально необходимая рабочая функциональность вкладок "Настройки", "Инструкции" и "Журнал".
вкладки «Анализ» и «Резерв» в рамках MVP должны быть реализованы как заглушки без бизнес-логики;
операции создания, переименования и удаления шаблонов должны быть доступны пользователю через интерфейс;
при возникновении компромисса между полнотой интерфейсной реализации и точностью логики обработки данных приоритет должен отдаваться точности логики обработки данных;
архитектура MVP должна предусматривать возможность последующего развития в сторону многопользовательского режима.
Вкладка «Журнал» должна быть полностью рабочей в UI.
Вкладка «Настройки» полноценная рабочая вкладка.
для типа «Дата» реализуем: ДД.ММ.ГГГГ, ГГГГ-ММ-ДД
для типа «Дата-время»: реализуем: ДД.ММ.ГГГГ ЧЧ:ММ, ГГГГ-ММ-ДД ЧЧ:ММ, ГГГГ-ММ-ДДTHH:MM:SS
Определение сессии : под сессией понимается период работы приложения от запуска до закрытия пользователем.
Шаблоны и пользовательские настройки в рамках MVP должны храниться как отдельные сущности.
Шаблон должен содержать поле версии.
Полноценная логика optimistic locking и предупреждение о параллельном изменении шаблона в MVP не реализуются и резервируются для будущей многопользовательской версии
8.	Архитектурные решения:

Держать три слоя данных:
исходный набор строк после загрузки файла;
рабочий набор после операций сессии, включая парсинг;
представление для UI после фильтрации, сортировки и скрытия столбцов.
Все фильтры, типизация и парсинг считать по полному рабочему набору, а лимит «Кол-во строк» использовать только как ограничение первичного отображения.
Ошибки хранить отдельно от строк таблицы: строка никогда не удаляется из-за ошибки типизации, а ячейка просто помечается, а ошибка попадает в нижнюю панель.
Парсинг делать как отдельную транзакцию с preview и snapshot для одного undo-шага.
Типы столбцов сохранять в шаблон сразу при изменении, без отдельной кнопки сохранения.
Для XLS/XLSX читать вычисленные значения формул, не формулы как текст.
Новый шаблон создается пользователем на основе структуры текущего загруженного файла.
