# Разбиение работы на спринты

## Спринт 1. Каркас системы
**Цель:** поднять основу проекта и зафиксировать архитектуру.

**Результат спринта:**
1. Создан backend на Python.
2. Создан frontend-каркас web-приложения.
3. Подключена SQLite.
4. Настроена базовая структура проекта: UI, API, слой хранения, слой обработки.
5. Реализована главная навигация по вкладкам.
6. Добавлены заглушки для вкладок Анализ, Инструкции, Журнал, Настройки, Резерв.
7. Подготовлена схема БД и миграции для templates, template_columns, app_settings, app_state, event_log.

**Критерий готовности:**
1. Приложение запускается локально.
2. Видны все основные вкладки.
3. БД создается автоматически.

## Спринт 2. Настройки, шаблоны, состояние интерфейса
**Цель:** собрать постоянную основу для работы пользователя.

**Результат спринта:**
1. Реализована вкладка Настройки.
2. Поддерживается ввод и валидация путей:
   - папка источника,
   - папка результата,
   - файл инструкций.
3. Реализован CRUD шаблонов.
4. Добавлено подтверждение удаления шаблона.
5. Реализовано хранение типов столбцов в шаблонах.
6. Реализовано сохранение состояния интерфейса между сессиями:
   - ширина панелей,
   - видимость правой панели,
   - видимость и размер панели Ошибки,
   - последний шаблон,
   - последний успешно загруженный файл.

**Критерий готовности:**
1. Настройки сохраняются и восстанавливаются.
2. Шаблоны можно создавать, переименовывать и удалять.
3. Состояние интерфейса переживает перезапуск приложения.

## Спринт 3. Загрузка фактов и базовое отображение
**Цель:** довести до рабочего состояния главный сценарий загрузки CSV.

**Результат спринта:**
1. Реализована вкладка Загрузка фактов.
2. Работает строка служебных сообщений.
3. Работает панель элементов.
4. Реализована загрузка CSV с правилами MVP:
   - UTF-8 with BOM,
   - разделитель ;
   - игнор пустых строк до заголовка,
   - обработка дубликатов имен столбцов.
5. Реализована таблица данных с виртуализацией строк.
6. Реализована нижняя панель Ошибки.
7. Реализована правая панель Настройка данных как контейнер с вкладками:
   - Общие,
   - Столбцы,
   - Фильтры,
   - Типы,
   - Обработка.
8. Реализована частичная загрузка с предупреждением и записью в журнал.

**Критерий готовности:**
1. Пользователь может выбрать шаблон, загрузить CSV и увидеть таблицу.
2. Ошибки и предупреждения отображаются.
3. Новые и отсутствующие столбцы обрабатываются по ТЗ.

## Спринт 4. Типы, ошибки, столбцы, фильтрация
**Цель:** реализовать основную логику управления данными.

**Результат спринта:**
1. Реализована вкладка Общие:
   - скрыть копейки,
   - только целые,
   - кол-во строк.
2. Реализована вкладка Столбцы:
   - видимость столбцов,
   - порядок столбцов.
3. Реализована вкладка Типы:
   - назначение типа,
   - автосохранение типа в шаблон,
   - пересчет ошибок по всем строкам.
4. Реализована вкладка Фильтры:
   - текст,
   - число,
   - денежный,
   - дата,
   - дата-время,
   - логический.
5. Реализована подсветка ошибочных ячеек.
6. Реализована сортировка по заголовку таблицы.
7. Реализована логика ограниченного отображения строк без потери полного рабочего набора.

**Критерий готовности:**
1. Типы влияют на валидацию и отображение.
2. Фильтры работают по всему набору.
3. Ошибки пересчитываются сразу.
4. Таблица корректно реагирует на скрытие, порядок и сортировку.

## Спринт 5. Обработка и сохранение результата
**Цель:** реализовать операции, изменяющие рабочий набор, и выгрузку результата.

**Результат спринта:**
1. Реализована вкладка Обработка.
2. Реализован режим Парсинг.
3. Реализован предпросмотр парсинга:
   - сколько строк удалится,
   - сколько строк создастся.
4. Реализовано выполнение парсинга по всем строкам после фильтрации.
5. Реализован откат последнего парсинга на один шаг.
6. Реализовано сохранение результата в XLSX.
7. Реализованы правила сохранения:
   - только видимые столбцы,
   - текущий порядок,
   - только строки после фильтрации,
   - отображаемые пользователю значения,
   - особое правило для денежного типа.
8. Реализована обработка ошибок сохранения.

**Критерий готовности:**
1. Пользователь может выполнить парсинг и откатить его.
2. Пользователь может сохранить результат в папку результата.
3. Имя файла формируется по правилам ТЗ.

**Критерий готовности:**
