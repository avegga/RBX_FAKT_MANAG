# 11.x. Панель комментариев к графику
- Рядом с каждым графиком отображается иконка комментариев (`icon_comments_{N}`) с индикатором количества новых/непрочитанных комментариев.
- При клике на иконку открывается панель комментариев (`panel_comments_{N}`), где отображается основной комментарий и ответы (один уровень вложенности, визуально отделены).
- В панели комментариев:
    - Список комментариев: каждый содержит текст (до 1000 знаков), автора, время создания, время последнего редактирования.
    - Для своих комментариев доступны кнопки «Редактировать» (`btn_edit_comment_{id}`) и «Удалить» (`btn_delete_comment_{id}`).
    - Для каждого комментария — кнопка «Ответить» (`btn_reply_comment_{id}`), ответ отображается с отступом справа.
    - При добавлении/редактировании комментария — автоматическое сохранение.
- Индикатор новых комментариев обновляется в реальном времени для всех пользователей.
- Панель комментариев закрывается по кнопке «Закрыть» или при клике вне панели.
# 11. Архитектурные принципы UI
Интерфейс вкладки "Анализ" должен быть реализован как самостоятельный модуль, максимально изолированный от UI и логики других вкладок приложения. Все взаимодействия с внешними данными, сервисами или общими компонентами должны осуществляться только через явно определённые интерфейсы или API. Внесение изменений в UI других вкладок при развитии функционала "Анализа" не допускается, за исключением случаев, предусмотренных архитектурой приложения.
Описание пользовательского интерфейса вкладки "Анализ"
========================================================

# 1. Общая структура интерфейса


Вкладка "Анализ" состоит из двух основных вертикальных областей, разделённых регулируемой вертикальной линейкой (разделителем):
- **Левая область** — конструктор анализа (панель управления графиками и настройками). Графики располагаются вертикально, один под другим. Для левой области предусмотрен вертикальный скроллбар при переполнении по высоте.
- **Правая область** — таблица данных для анализа (отображает строки и столбцы, участвующие в построении графиков). Для правой области предусмотрены вертикальный и горизонтальный скроллбары для просмотра больших таблиц.

Положение разделителя между областями регулируется пользователем, ширина сохраняется между сессиями.

Если графики строятся на основе разных таблиц, для каждого графика отображается соответствующая таблица напротив него (график и таблица располагаются в одной горизонтальной линии). В обычном режиме — одна общая таблица для всех графиков.

Для каждого графика и для всех графиков одновременно предусмотрена возможность печати в файл (PDF, PNG, SVG — на выбор пользователя).

Каждый график можно развернуть в полноэкранный режим по отдельности. Все четыре графика можно развернуть одновременно в отдельное полноэкранное окно.

# 2. Верхняя панель управления (toolbar)

В верхней части вкладки располагается панель управления с основными кнопками и выпадающими списками:

- **Кнопка "Загрузить файл"** (`btn_upload_file`): открывает диалог выбора файла для загрузки данных анализа.
- **Кнопка "Использовать загрузку фактов"** (`btn_use_facts`): загружает данные из вкладки "Загрузка фактов".
- **Выпадающий список "Тип анализа"** (`select_analysis_type`): выбор или создание типа анализа (сохранённой конфигурации графиков).
- **Кнопка "Создать тип анализа"** (`btn_create_analysis_type`): открывает диалог создания нового типа анализа.
- **Кнопка "Удалить тип анализа"** (`btn_delete_analysis_type`): удаляет выбранный тип анализа.
- **Кнопка "Сохранить тип анализа"** (`btn_save_analysis_type`): сохраняет изменения текущего типа анализа.
- **Кнопка "Сбросить настройки"** (`btn_reset_settings`): сбрасывает настройки текущего анализа к значениям по умолчанию.

# 3. Левая область: конструктор анализа


В левой области располагается панель управления графиками и их параметрами. Для каждого графика отображается отдельная карточка с настройками:

- **Список графиков** (`list_charts`): отображает все графики текущего типа анализа (максимум 4).
- **Кнопка "Добавить график"** (`btn_add_chart`): добавляет новый график.
- **Карточка графика** (`chart_card_{N}`):
    - **Выпадающий список "Тип графика"** (`select_chart_type_{N}`): выбор типа (столбиковая диаграмма, линейный график, круговая диаграмма, таблица-агрегация).
    - **Выпадающие списки для выбора полей** (`select_x_{N}`, `select_y_{N}`, `select_group_{N}`, `select_agg_{N}`): выбор оси X, оси Y, группировки, агрегации.
    - **Параметры визуализации** (`input_color_{N}`, `input_legend_{N}`, `input_labels_{N}`): цвет, легенда, подписи.
    - **Кнопка "Удалить график"** (`btn_delete_chart_{N}`): удаляет график.
    - **Кнопка "Скрыть график"** (`btn_hide_chart_{N}`): временно скрывает график.
    - **Кнопка "Печать графика"** (`btn_print_chart_{N}`): открывает диалог печати для выбранного графика.
    - **Drag&Drop-зона** (`drag_chart_{N}`): позволяет менять порядок и расположение графиков.

# 3.1. Печать графиков

- **Кнопка "Печать всех графиков"** (`btn_print_all_charts`): расположена на панели управления графиками или в верхней панели.
- При нажатии кнопки печати (на карточке графика или для всех графиков) открывается диалог выбора формата (PDF, PNG, SVG).
- Пользователь выбирает формат и подтверждает действие (кнопка "Сохранить" или "Печать").
- Формируется файл с изображением выбранного графика или всех графиков в выбранном формате.
- Пользователь выбирает место сохранения файла.
- Если график скрыт, кнопка печати для него неактивна.

# 4. Правая область: таблица данных анализа

- **Таблица данных** (`data_table`): отображает строки и столбцы, участвующие в анализе.
- **Строка поиска** (`input_search`): поиск по данным таблицы.
- **Панель фильтрации** (`panel_filters`): фильтрация по столбцам.
- **Панель сортировки** (`panel_sorting`): сортировка по столбцам.
- **Индикатор количества строк/столбцов** (`label_count`): отображает текущее количество строк и столбцов.

# 5. Вспомогательные элементы и состояния

- **Сообщения и предупреждения** (`alert_message`): отображаются в случае ошибок загрузки, несовместимости данных, отсутствия данных и других ситуаций.
- **Пустое состояние** (`empty_state`): отображается, если нет выбранного источника данных или нет данных для анализа.
- **Индикатор загрузки** (`loading_indicator`): отображается при загрузке или обработке данных.

# 6. Взаимное расположение элементов

- Верхняя панель управления занимает всю ширину вкладки и располагается сверху.
- Под ней — две основные области: левая (конструктор анализа) и правая (таблица данных), разделённые вертикальным разделителем (`splitter`).
- Левая область фиксированной минимальной ширины, правая — адаптивная.
- В левой области сверху — кнопки управления графиками, ниже — список карточек графиков.
- В правой области сверху — строка поиска, панели фильтрации и сортировки, ниже — таблица данных.
- Сообщения, предупреждения и индикаторы отображаются поверх соответствующих областей.

# 7. Именование элементов для разработки

Все элементы снабжены уникальными идентификаторами (см. в скобках выше) для использования в коде и автотестах.

# 8. Особенности UX

- Все действия пользователя (добавление, удаление, скрытие, перемещение графиков, изменение параметров, загрузка данных) должны быть доступны с клавиатуры.
- Интерфейс поддерживает масштабирование и локализацию (ru_RU, en_US).
- Все предупреждения и ошибки отображаются максимально явно и не блокируют работу пользователя.
- Размеры областей, выбранные типы анализа и другие пользовательские настройки сохраняются между сессиями.

# 9. Сценарии отображения

- Если источник данных не выбран — отображается пустое состояние с предложением выбрать или загрузить данные.
- Если после фильтрации нет строк — отображается предупреждение.
- Если график несовместим с данными — карточка графика выделяется и отображается сообщение с рекомендацией.
- При ошибке загрузки — отображается сообщение и кнопка повторить попытку.

# 10. Прочее

- Максимальное количество графиков на экране — 4.
- Все действия пользователя должны быть интуитивно понятны и сопровождаться подсказками.
- Все элементы должны быть доступны для автотестирования по id.
