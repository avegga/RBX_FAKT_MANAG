# 15. Комментарии к графикам (выводы)
1. Для каждого графика во вкладке "Анализ" предусмотрена возможность добавления, просмотра, редактирования и удаления комментариев (выводов).
2. Рядом с каждым графиком отображается иконка комментариев с индикатором количества новых/непрочитанных комментариев.
3. При клике на иконку открывается панель комментариев, где отображается основной комментарий и ответы к нему (один уровень вложенности, визуально отделены).
4. Каждый комментарий содержит: текст (до 1000 знаков), автора, время создания, время последнего редактирования.
5. Пользователь может редактировать и удалять только свои комментарии.
6. Добавление и редактирование комментариев сопровождается автоматическим сохранением.
7. Все пользователи видят всю историю комментариев по графику.
8. При появлении новых комментариев или ответов индикатор на иконке обновляется для всех пользователей.
9. История комментариев хранится в базе данных и связана с типом анализа и конкретным графиком.
10. Поиск и фильтрация по комментариям реализуются по мере необходимости.

# 1. Правила построения анализа
Графики и аналитические представления строятся только по видимым строкам правой таблицы. Режим построения по полному набору строк не используется.
Если источником данных является вкладка "Загрузка фактов", в анализ передаются данные после фильтров из вкладки "Загрузка фактов". После передачи этих данных пользователь может дополнительно применять сортировку, фильтрацию и поиск уже внутри вкладки "Анализ" — итоговый набор видимых строк используется для расчёта графиков.

# 2. Левая область: конструктор анализа
Левая область содержит конструктор графиков и аналитических представлений. Набор элементов управления меняется в зависимости от типа графика.

# 3. Правая область: таблица данных
Правая область содержит таблицу данных, участвующих в расчёте графиков.

# 4. Количество и управление графиками
На одном экране допускается не более 4 графиков одновременно. Для каждого графика сохраняются:
- идентификатор в составе типа анализа
- тип графика
- выбранные поля
- параметры визуализации и агрегации
- порядок отображения
- координаты на экране
- признак совместимости с текущим источником данных

# 4. Пустые состояния и предупреждения
Во вкладке "Анализ" все пустые состояния и предупреждения отображаются в левой области — в области построения графиков. Сценарии:
1. Источник данных не выбран — графики не строятся, показывается предупреждение.
2. Во вкладке "Загрузка фактов" отсутствуют данные — при попытке использовать этот источник графики не строятся, показывается предупреждение.
3. После фильтров во вкладке "Загрузка фактов" нет строк — графики не строятся, показывается предупреждение.
4. Ошибка загрузки файла анализа — графики не строятся, показывается предупреждение.

# 5. Рабочая область

Рабочая область разделена вертикальной линейкой на две части. Линейка позволяет изменять размеры областей по ширине, положение разделителя сохраняется между сессиями.
# 5.1. Левая область
- В левой области отображаются все пустые состояния и предупреждения, связанные с анализом (см. п. 4).
Для каждого графика пользователь может независимо выбрать источник данных (таблицу) из доступных вариантов с помощью отдельного элемента управления (например, выпадающего списка) в настройках графика. После выбора таблицы для графика в правой области автоматически отображается соответствующая таблица данных, используемая для построения этого графика. Выбор источника данных для каждого графика сохраняется в конфигурации анализа и может быть изменён пользователем в любой момент.
Левая область содержит:
- конструктор графиков и аналитических представлений (набор элементов управления меняется в зависимости от типа графика);
- панель управления графиками;
- обязательные типы представлений: столбиковая диаграмма, линейный график, круговая диаграмма, таблица-агрегация;
- графики располагаются вертикально, один под другим;
- вертикальный скроллбар при переполнении по высоте;
- для каждого графика предусмотрена возможность развернуть его в полноэкранный режим (fullscreen), а также развернуть все четыре графика одновременно в отдельное полноэкранное окно.

# 5.2. Правая область
В правой области может отображаться несколько таблиц — для каждого графика отдельная, если выбраны разные источники данных. В обычном режиме отображается одна общая таблица для всех графиков.
Содержит таблицу данных, участвующих в расчёте графиков:
- Предусмотрены вертикальный и горизонтальный скроллбары для просмотра больших таблиц.
- Если графики строятся на основе разных таблиц, для каждого графика отображается соответствующая таблица напротив него (график и таблица располагаются в одной горизонтальной линии). В обычном режиме — одна общая таблица для всех графиков.

**Печать графиков:**
- Для каждого графика и для всех графиков одновременно предусмотрена возможность печати в файл (PDF, PNG, SVG — на выбор пользователя).

**Полноэкранный режим:**
- Каждый график можно развернуть в полноэкранный режим по отдельности.
- Все четыре графика можно развернуть одновременно в отдельное полноэкранное окно.

# 6. Повторная загрузка данных
1. Повторная загрузка файла во вкладке "Анализ": если пользователь повторно нажимает кнопку "Загрузить файл" и выбирает другой файл, ранее загруженный набор данных анализа полностью заменяется новым. Сохранённая конфигурация типа анализа не удаляется. Если новый файл несовместим с графиками, система показывает предупреждение, несовместимые графики не строятся до переназначения полей.
2. Повторное использование данных из "Загрузка фактов": аналогично — набор данных заменяется, несовместимые графики не строятся.

# 7. Правила обновления анализа
**Анализ пересчитывается автоматически:**
- при изменении сортировки, фильтрации, поиска в правой таблице
- при изменении параметров графика
- при изменении состава или расположения графиков

**Анализ обновляется только по кнопке:**
- при первом подключении файла анализа
- при получении/повторном получении данных из "Загрузка фактов"

**Реакция на изменения во вкладке "Загрузка фактов":**
Если пользователь изменил данные, фильтры, сортировку или иные параметры во вкладке "Загрузка фактов", вкладка "Анализ" не обновляется автоматически. Для обновления нужно повторно нажать кнопку "Использовать загрузку фактов".

**Реакция на несовместимость источника данных:**
Если новый источник несовместим с графиками, система показывает предупреждение, несовместимые графики не строятся, тип анализа и расположение графиков сохраняются.

# 8. Сохранение состояния между сессиями
**В типе анализа хранятся:**
- имя типа анализа
- набор графиков
- типы графиков
- параметры графиков
- выбранные поля
Если новый источник несовместим с графиками, система показывает предупреждение, несовместимые графики не строятся, тип анализа и расположение графиков сохраняются.

# 8. Сохранение состояния между сессиями
**В типе анализа хранятся:**
- имя типа анализа
- набор графиков
- типы графиков
- параметры графиков
- выбранные поля
- параметры агрегации, группировки, легенды, цвета, подписей
- порядок и координаты графиков
- признаки совместимости графиков с текущим набором данных

**В пользовательском состоянии хранятся:**
- последний выбранный тип анализа
- последний выбранный источник данных
- размеры левой и правой областей
- визуально восстановленное состояние интерфейса

**Восстановление после перезапуска:**
После запуска приложения последний выбранный тип анализа и источник данных восстанавливаются визуально, но источник не активируется автоматически — пользователь должен явно нажать кнопку.

# 9. Журналирование
Во вкладке "Анализ" журналирование не выполняется. События вкладки (построение графиков, ошибки загрузки, несовместимость полей, переключение источника, создание/изменение типа анализа) в журнал не записываются.

# 10. Хранение
Список типов анализа и их конфигурации, а также пользовательское состояние вкладки "Анализ" хранятся в SQLite. При изменении структуры или обновлении приложения обеспечивается обратная совместимость и миграция данных.

# 11. Дополнительные требования и уточнения
## 11.1. Ограничения по объёму данных
Вкладка "Анализ" должна стабильно работать с наборами до 50 000 строк и 100 столбцов. При превышении этих значений допускается предупреждение о возможном снижении производительности.

## 11.2. Поддерживаемые типы данных
Поддерживаются: число, текст, дата, булево. Некорректные или смешанные типы помечаются ошибкой и не используются в расчётах графиков.

## 11.3. Поведение при удалении типа анализа
Если последний выбранный тип анализа был удалён, при следующем запуске вкладка сбрасывается к пустому состоянию с предложением создать новый тип анализа.

## 11.4. UX при несовместимости графика и данных
Если график не может быть построен из-за несовместимости полей или типов, пользователю выводится подробное сообщение с причиной и рекомендацией по исправлению.

## 11.5. Управление графиками на экране
Пользователь может временно скрывать, удалять и менять местами графики drag&drop. Все изменения порядка и координат сохраняются. Скрытые графики не участвуют в расчётах, но их конфигурация сохраняется.

## 11.6. Доступность и локализация
Интерфейс поддерживает масштабирование, работу с клавиатурой, локализацию сообщений и форматов дат/чисел (ru_RU и en_US минимум).

## 11.7. Формат хранения и миграция
Все пользовательские данные и состояния хранятся в SQLite. При изменении структуры или обновлении приложения обеспечивается обратная совместимость и миграция данных без потери пользовательских настроек.

## 11.8. Минимальные требования к тестированию
Критичные сценарии для тестирования:
- быстрая смена источников данных
- загрузка больших файлов
- удаление всех графиков
- повторная загрузка несовместимых данных
- восстановление состояния после сбоя

## 11.9. Поведение при ошибках загрузки данных
При ошибке загрузки пользователь может повторить попытку. После неудачной загрузки состояние вкладки очищается до выбора нового источника.

## 11.10. Интеграция с другими вкладками
Если во вкладке "Загрузка фактов" или в шаблонах происходят изменения, не влияющие на текущий набор данных анализа, вкладка "Анализ" не обновляется автоматически. Если выбранный источник или шаблон был удалён, пользователю показывается предупреждение и предлагается выбрать новый источник или создать новый тип анализа.
- не строить несовместимые графики;
- сохранить сам тип анализа;
- сохранить расположение графиков;
- ожидать от пользователя переназначения полей или изменения параметров.
