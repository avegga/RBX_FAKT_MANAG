# Результаты разработки по спринтам

## Спринт 1. Каркас системы

### Выполненные работы

1. Настроен backend на Python с использованием Flask.
2. Создан каркас frontend-приложения на HTML, CSS и JavaScript.
3. Интегрирована база данных SQLite.
4. Определена структура проекта: UI, API, хранилище и слой обработки.
5. Реализована основная навигация по вкладкам.
6. Добавлены базовые заглушки для разделов:
   - Анализ;
   - Инструкции;
   - Журнал;
   - Настройки;
   - Резерв.
7. Подготовлена схема базы данных и выполнена начальная инициализация.

### Результат

Первый спринт закрыл задачи по созданию базового каркаса приложения. Проект был подготовлен к переходу на рабочую реализацию пользовательских настроек и шаблонов.

## Спринт 2. Настройки, шаблоны, состояние интерфейса

### Выполненные работы

1. Переработан Flask backend: приложение теперь отдаёт полноценный web-интерфейс и JSON API.
2. Реализована вкладка Настройки с сохранением следующих параметров:
   - папка источника;
   - папка результата;
   - файл инструкций.
3. Добавлена серверная валидация путей:
   - путь к папке должен существовать и быть директорией;
   - путь к файлу инструкций должен существовать и быть файлом.
4. Реализован CRUD шаблонов:
   - создание шаблона;
   - переименование шаблона;
   - удаление шаблона с подтверждением;
   - хранение состава столбцов шаблона.
5. Добавлено хранение типов столбцов шаблона:
   - text;
   - number;
   - money;
   - date;
   - datetime;
   - boolean.
6. Реализовано сохранение состояния интерфейса между сессиями:
   - ширина левой панели;
   - видимость правой панели;
   - видимость панели ошибок;
   - высота панели ошибок;
   - последний выбранный шаблон;
   - последний успешно загруженный файл.
7. Добавлен журнал событий приложения для операций с настройками и шаблонами.
8. Обновлены SQL-схема и скрипт инициализации базы данных.
9. Реализован обновлённый интерфейс второго спринта с рабочими формами, вкладками и связью с backend API.

### Проверка

1. Схема БД инициализируется через `python init_db.py`.
2. Приложение запускается локально через `python app.py`.
3. Главная страница приложения отвечает со статусом 200.
4. API `/api/bootstrap` возвращает корректную структуру настроек, шаблонов и состояния интерфейса.

### Результат

Второй спринт перевёл проект от каркаса к рабочему MVP-уровню для базовой пользовательской настройки. Следующий логичный шаг разработки — загрузка CSV, таблица данных и полноценная работа панели ошибок в рамках третьего спринта.
