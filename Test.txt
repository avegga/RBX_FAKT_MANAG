# Тестирование

## Спринт 1

- [ ] Верхняя панель управления отображается по всей ширине экрана, содержит все заявленные кнопки и выпадающие списки:
	- [ ] Кнопка "Загрузить файл"
	- [ ] Кнопка "Использовать загрузку фактов"
	- [ ] Выпадающий список "Тип анализа"
	- [ ] Кнопка "Создать тип анализа"
	- [ ] Кнопка "Удалить тип анализа"
	- [ ] Кнопка "Сохранить тип анализа"
	- [ ] Кнопка "Сбросить настройки"
- [ ] Левая область содержит заголовок/иконку панели, список (заглушки) карточек графиков (до 4), кнопку "Добавить график".
- [ ] Каждая карточка графика имеет место под название, тип, параметры (пусть пока это просто заглушки).
- [ ] Правая область содержит заголовок/иконку таблицы, заглушку для таблицы данных с полосами прокрутки, место под строку поиска, фильтры, сортировку, индикатор количества строк/столбцов.
- [ ] Вертикальный разделитель между областями работает: его можно перетаскивать, ширина областей меняется.
- [ ] Все элементы интерфейса размещены максимально компактно: нет лишних отступов, пустых областей, всё помещается даже при уменьшении ширины окна.
- [ ] Минимальные размеры областей позволяют работать даже на небольших экранах.
- [ ] Стилизация соответствует современным стандартам: цвета, шрифты, отступы выглядят аккуратно и не делают интерфейс громоздким.
- [ ] Заглушки для будущих элементов (например, иконки комментариев, печати) присутствуют и не мешают восприятию основной структуры.
- [ ] При уменьшении окна интерфейс остаётся удобным, не появляется горизонтальный скролл.
- [ ] Все элементы видимы, не перекрываются, не выходят за границы экрана.

## Спринт 2

- [ ] Верхняя панель вкладки "Анализ" отображается и содержит рабочие элементы управления:
	- [ ] Кнопка "Загрузить файл"
	- [ ] Кнопка "Использовать загрузку фактов"
	- [ ] Выпадающий список "Тип анализа"
	- [ ] Кнопка "Создать тип анализа"
	- [ ] Кнопка "Удалить тип анализа"
	- [ ] Кнопка "Сохранить тип анализа"
	- [ ] Кнопка "Сбросить настройки"
- [ ] При открытии вкладки "Анализ" отображается корректное начальное сообщение, если тип анализа и источник данных еще не выбраны.
- [ ] Кнопка "Создать тип анализа" создает новый тип анализа, после чего он появляется в выпадающем списке.
- [ ] Выпадающий список "Тип анализа" позволяет выбрать ранее созданный тип анализа.
- [ ] Кнопка "Удалить тип анализа" удаляет выбранный тип анализа после подтверждения.
- [ ] Кнопка "Сохранить тип анализа" сохраняет изменения текущего типа анализа.
- [ ] Кнопка "Сбросить настройки" сбрасывает локальные настройки текущего типа анализа без поломки интерфейса.
- [ ] Кнопка "Использовать загрузку фактов" подключает данные из вкладки "Загрузка фактов", если там есть активный набор данных.
- [ ] Если во вкладке "Загрузка фактов" нет данных, при подключении отображается понятное сообщение об ошибке.
- [ ] Левая область "Анализ" позволяет добавить карточки графиков кнопкой "Добавить график".
- [ ] Нельзя добавить более 4 графиков одновременно.
- [ ] Каждая карточка графика содержит рабочие поля:
	- [ ] Тип графика
	- [ ] Источник
	- [ ] X
	- [ ] Y
	- [ ] Group
	- [ ] Agg
	- [ ] Цвет
	- [ ] Легенда
	- [ ] Подписи
- [ ] Для каждой карточки графика работают действия:
	- [ ] Скрыть/показать
	- [ ] Удалить
	- [ ] Переместить вверх
	- [ ] Переместить вниз
- [ ] После изменения порядка графиков порядок визуально обновляется корректно.
- [ ] После удаления графика нумерация и элементы управления остаются корректными.
- [ ] Правая область отображает таблицу данных анализа после подключения источника.
- [ ] В таблице отображаются строки и столбцы из подключенного источника данных.
- [ ] Строка поиска фильтрует таблицу по содержимому строк.
- [ ] Сортировка по заголовку столбца работает и меняет порядок строк.
- [ ] Кнопка сброса сортировки возвращает таблицу в исходное состояние сортировки.
- [ ] Индикатор количества строк и столбцов показывает актуальные значения.
- [ ] Если после поиска строки отсутствуют, отображается отдельное предупреждающее состояние.
- [ ] Если источник данных не подключен, отображается empty state вместо таблицы.
- [ ] Во время загрузки данных анализа отображается loading state.
- [ ] При ошибке подключения данных или загрузки состояния отображается error state или сообщение об ошибке.
- [ ] Кнопка повтора подключения данных в empty/error state работает корректно.
- [ ] Вертикальный разделитель между левой и правой областями работает: его можно перетаскивать, и ширина левой панели меняется.
- [ ] После перезапуска приложения сохраняются:
	- [ ] Последний выбранный тип анализа
	- [ ] Ширина левой панели вкладки "Анализ"
	- [ ] Последний подключенный визуальный источник данных
- [ ] После перезапуска приложения ранее сохраненные карточки графиков загружаются для выбранного типа анализа.
- [ ] Все основные элементы вкладки "Анализ" имеют стабильные id и доступны для автотестов.
- [ ] Интерфейс вкладки "Анализ" не ломает существующую работу вкладок "Загрузка фактов", "Настройки", "Инструкции" и "Журнал".

## Спринт 3

- [ ] Кнопка "Загрузить файл" во вкладке "Анализ" открывает сценарий загрузки отдельного аналитического файла.
- [ ] Вкладка "Анализ" принимает отдельный файл анализа как источник данных.
- [ ] После загрузки отдельного файла анализа правая таблица заполняется его данными.
- [ ] Кнопка подключения данных из вкладки "Загрузка фактов" использует не прямой внутренний набор, а отдельный экспортируемый слой данных.
- [ ] Экспортируемый слой из вкладки "Загрузка фактов" формируется корректно и может быть использован как источник Анализа.
- [ ] При подключении экспортируемого слоя из вкладки "Загрузка фактов" правая таблица Анализа показывает данные именно этого слоя.
- [ ] Пользователь видит, какой источник данных сейчас активен:
	- [ ] данные из вкладки "Загрузка фактов"
	- [ ] отдельно загруженный файл анализа
- [ ] В интерфейсе отображаются основные метаданные активного источника:
	- [ ] тип источника
	- [ ] имя файла или имя набора
	- [ ] время подключения
- [ ] Пользователь может заменить текущий источник Анализа другим источником без поломки вкладки.
- [ ] При смене источника правая таблица Анализа корректно обновляется.
- [ ] Если структура нового источника несовместима с сохраненным типом анализа, пользователь получает понятное предупреждение.
- [ ] При несовместимости источника конфигурация графиков не теряется автоматически.
- [ ] Карточки графиков, требующие перенастройки из-за смены источника, явно помечаются в интерфейсе.
- [ ] Если выбранный файл анализа недоступен или поврежден, отображается понятное сообщение об ошибке.
- [ ] При ошибке загрузки доступна повторная попытка загрузки или подключения источника.
- [ ] Если источник не выбран, отображается корректное empty state.
- [ ] Если источник загружается, отображается loading state.
- [ ] После успешной загрузки отдельного файла или подключения экспортируемого слоя индикаторы строк и столбцов отображают корректные значения.
- [ ] Правая таблица Анализа одинаково работает для двух видов источников:
	- [ ] отдельный файл анализа
	- [ ] экспортируемый слой из вкладки "Загрузка фактов"
- [ ] Поиск по таблице Анализа работает для отдельного файла анализа.
- [ ] Поиск по таблице Анализа работает для экспортируемого слоя из вкладки "Загрузка фактов".
- [ ] Сортировка по столбцам работает для отдельного файла анализа.
- [ ] Сортировка по столбцам работает для экспортируемого слоя из вкладки "Загрузка фактов".
- [ ] После перезапуска приложения восстанавливаются:
	- [ ] последний активный источник Анализа
	- [ ] метаданные активного источника
	- [ ] рабочая таблица для этого источника
- [ ] Если после перезапуска ранее выбранный файл анализа недоступен, приложение показывает корректное сообщение и не ломает вкладку.
- [ ] Если после перезапуска экспортируемый слой из вкладки "Загрузка фактов" недоступен, приложение показывает корректное сообщение и предлагает повторное подключение.
- [ ] Работа с новыми источниками не ломает существующие функции Спринта 2:
	- [ ] типы анализа
	- [ ] карточки графиков
	- [ ] поиск
	- [ ] сортировку
	- [ ] сохранение состояния вкладки

## Спринт 4

- [ ] Во вкладке "Анализ" для карточки графика появляется реальная область визуализации, а не только набор полей настройки.
- [ ] Пользователь может построить как минимум базовые типы графиков, заявленные в спринте:
	- [ ] столбчатая диаграмма
	- [ ] линейный график
	- [ ] третий базовый тип графика, если он включен в реализацию спринта
- [ ] Построение графика работает для активного источника Анализа.
- [ ] Поле X участвует в построении графика и влияет на результат.
- [ ] Поле Y участвует в построении графика и влияет на результат.
- [ ] Поле Group, если выбрано, влияет на группировку данных в графике.
- [ ] Поле Agg, если выбрано, применяет корректную агрегацию к данным.
- [ ] Если в реализации используются color, legend и labels, эти параметры реально влияют на отображение графика.
- [ ] Для поддерживаемых агрегаций корректно работают как минимум:
	- [ ] count
	- [ ] sum
	- [ ] avg
- [ ] Если для графика не выбраны обязательные параметры, вместо сломанного графика отображается понятное состояние "недостаточно параметров" или аналогичное сообщение.
- [ ] Если график невозможно построить из-за несовместимых полей, отображается понятная ошибка или предупреждение.
- [ ] Если по выбранной конфигурации нет данных для отображения, показывается отдельное состояние "нет данных".
- [ ] После корректной настройки карточки график успешно строится без перезагрузки страницы.
- [ ] При изменении параметров карточки график корректно перестраивается.
- [ ] При смене типа графика визуализация меняется корректно и без поломки карточки.
- [ ] Если источник данных сменился и карточка стала несовместимой, вместо графика показывается понятное сообщение о необходимости перенастройки.
- [ ] При несовместимости источника сохраненная конфигурация карточки не теряется автоматически.
- [ ] Графики работают как минимум с двумя типами источников Анализа:
	- [ ] отдельно загруженный файл анализа
	- [ ] экспортируемый слой из вкладки "Загрузка фактов"
- [ ] При переключении между источниками график либо корректно перестраивается, либо показывает понятную причину, почему построение невозможно.
- [ ] Ошибка построения одного графика не ломает остальные карточки и не ломает вкладку "Анализ" целиком.
- [ ] Если карточка скрыта, это не приводит к поломке состояния или визуализации остальных графиков.
- [ ] После сохранения типа анализа и перезапуска приложения конфигурация карточек сохраняется и может быть повторно использована для построения графиков.
- [ ] После повторного открытия приложения ранее рабочая конфигурация графика восстанавливается предсказуемо.
- [ ] Построение графиков не ломает функциональность Спринта 2:
	- [ ] создание и выбор типа анализа
	- [ ] сохранение типа анализа
	- [ ] добавление и удаление карточек
	- [ ] изменение порядка карточек
	- [ ] поиск по таблице
	- [ ] сортировку таблицы
- [ ] Построение графиков не ломает функциональность Спринта 3:
	- [ ] загрузку отдельного файла анализа
	- [ ] подключение экспортируемого слоя из вкладки "Загрузка фактов"
	- [ ] отображение метаданных источника
	- [ ] предупреждения о несовместимости
	- [ ] восстановление активного источника после перезапуска

## Спринт 5

- [ ] Для карточки графика доступны действия экспорта и/или печати результата.
- [ ] Пользователь может сохранить результат текущего графика в файл.
- [ ] Пользователь может распечатать текущий график.
- [ ] Если для карточки доступна агрегированная таблица, пользователь может выгрузить и ее.
- [ ] В экспорт или печатное представление попадает именно тот результат, который пользователь видит на экране.
- [ ] В печатной или экспортной форме отображаются базовые метаданные результата:
	- [ ] название графика или карточки
	- [ ] тип графика
	- [ ] источник данных
	- [ ] дата или время формирования
- [ ] Если у графика заданы legend и labels, они корректно учитываются в экспорте или печати.
- [ ] Пользователь может выгрузить только активный график.
- [ ] Если сценарий спринта включает пакетную выгрузку, пользователь может выгрузить все видимые графики текущего типа анализа.
- [ ] Если график не настроен, вместо ошибочного экспорта отображается понятное сообщение, что экспорт недоступен.
- [ ] Если источник данных недоступен, экспорт и печать не ломают вкладку и показывают понятное сообщение об ошибке.
- [ ] Если по конфигурации графика нет данных, экспорт не создает некорректный результат и показывает понятное состояние.
- [ ] Экспорт или печать одного графика не ломают остальные карточки и не ломают вкладку "Анализ" целиком.
- [ ] После экспорта или печати конфигурация карточки графика не сбрасывается.
- [ ] Экспорт не ломает построение графиков из Спринта 4:
	- [ ] столбчатая диаграмма
	- [ ] линейный график
	- [ ] круговая диаграмма или другой третий базовый тип, если он реализован
	- [ ] табличная агрегация
- [ ] Экспорт и печать не ломают функциональность Спринта 3:
	- [ ] загрузку отдельного файла анализа
	- [ ] подключение экспортируемого слоя из вкладки "Загрузка фактов"
	- [ ] отображение метаданных источника
	- [ ] переключение между источниками
- [ ] Экспорт и печать не ломают функциональность Спринта 2:
	- [ ] выбор типа анализа
	- [ ] сохранение типа анализа
	- [ ] карточки графиков
	- [ ] поиск по таблице
	- [ ] сортировку таблицы
- [ ] После перезапуска приложения сценарии экспорта и печати остаются доступны для ранее сохраненной конфигурации, если источник данных доступен.
