# SPRINTS

## Спринт 1. Базовый пользовательский интерфейс


**Цель:**
Создать визуальный каркас вкладки "Анализ" с акцентом на компактность и современный стиль, чтобы пользователь сразу увидел структуру и основные элементы интерфейса.

**В рамках спринта будут реализованы:**
- Верхняя панель управления (toolbar) по всей ширине экрана, включающая:
	- Кнопку "Загрузить файл"
	- Кнопку "Использовать загрузку фактов"
	- Выпадающий список "Тип анализа"
	- Кнопки "Создать", "Удалить", "Сохранить тип анализа", "Сбросить настройки"
- Левая область (панель управления графиками):
	- Заголовок или иконка панели
	- Список (заглушки) карточек графиков (до 4), каждая с местом под название, тип, параметры
	- Кнопка "Добавить график"
- Правая область (таблица данных):
	- Заголовок или иконка таблицы
	- Заглушка для таблицы данных с полосами прокрутки
	- Место под строку поиска, фильтры, сортировку, индикатор количества строк/столбцов
- Вертикальный разделитель между областями с возможностью перетаскивания
- Базовая стилизация: компактные отступы, современная цветовая схема, адаптивная ширина
- Минимальные размеры областей, чтобы интерфейс оставался удобным даже на небольших экранах
- Заглушки для будущих элементов (например, иконки комментариев, печати и т.д.)

**Цитаты и формулировки из проектных файлов**

**ТЗ Анализ V1.txt:**
- «Вкладка "Анализ" состоит из двух основных вертикальных областей, разделённых регулируемой вертикальной линейкой (разделителем)...»
- «Верхняя панель управления с основными кнопками и выпадающими списками..."
- (Добавить из других вкладок, например:) «Требование: все элементы интерфейса должны быть максимально компактными, чтобы обеспечить удобство работы даже на небольших экранах.»

**Описание UI.txt:**

- «Верхняя панель управления (toolbar)...»
- «В левой области располагается панель управления графиками и их параметрами..."

## Спринт 3. Загрузка и выбор данных

**Цель:**
Сделать вкладку "Анализ" полноценной точкой выбора источника данных: поддержать загрузку отдельного аналитического файла, подключение данных из вкладки "Загрузка фактов" и закрепить единый слой данных, который передается в Анализ.

**Ключевое архитектурное решение спринта:**
- Во вкладку "Анализ" из вкладки "Загрузка фактов" передается не прямой snapshot текущего рабочего набора и не snapshot UI-представления после фильтрации.
- В Анализ передается отдельный специально подготовленный экспортируемый слой данных.
- Этот слой должен быть стабильным, предсказуемым и независимым от временного UI-состояния вкладки "Загрузка фактов".
- Экспортируемый слой становится официальным источником для аналитики, дальнейшего построения графиков и повторного использования между сессиями.

**Backend-задачи:**
- Добавить модель источника данных Анализа
- Разделить источники Анализа на:
	- данные из вкладки "Загрузка фактов"
	- отдельно загруженный файл анализа
- Реализовать подготовку экспортируемого слоя данных из вкладки "Загрузка фактов"
- Реализовать API для:
	- загрузки отдельного файла анализа
	- подключения экспортируемого слоя из вкладки "Загрузка фактов"
	- получения текущего активного источника Анализа
	- очистки или замены источника Анализа
- Сохранять метаданные источника:
	- тип источника
	- имя файла или имя набора
	- время подключения
	- признак доступности
	- структуру столбцов

**Frontend-задачи:**
- Сделать рабочей кнопку "Загрузить файл" во вкладке "Анализ"
- Показывать в UI текущий активный источник данных Анализа
- Дать пользователю возможность переключаться между:
	- экспортируемым слоем из вкладки "Загрузка фактов"
	- отдельным файлом анализа
- Обновить правую таблицу так, чтобы она работала с обоими типами источников
- Показывать понятные состояния:
	- источник не выбран
	- файл загружается
	- файл загружен
	- источник недоступен
	- структура источника несовместима с сохраненным типом анализа
- Добавить сценарий повторной попытки при ошибке загрузки или подключения источника

**Работа со слоем данных для Анализа:**
- Экспортируемый слой из вкладки "Загрузка фактов" должен быть явной и отдельной сущностью
- Этот слой не должен зависеть от временных пользовательских действий на уровне интерфейса, если они не входят в согласованное правило экспорта
- В рамках спринта нужно закрепить и реализовать единый контракт экспортируемого слоя:
	- список столбцов
	- порядок столбцов
	- строки набора
	- технические метаданные источника
- Анализ должен читать только этот слой, а не обращаться напрямую к внутренним структурам вкладки "Загрузка фактов"

**Совместимость типа анализа и источника:**
- Если сохраненный тип анализа ссылается на поля, которых нет в новом источнике, это должно явно отображаться в UI
- Карточки графиков должны помечаться как требующие перенастройки
- При этом конфигурация не должна теряться автоматически

**Изменения в БД:**
- Таблица источников данных Анализа
- Хранение последнего активного источника Анализа
- Хранение метаданных экспортируемого слоя
- При необходимости отдельное хранение подготовленного слоя для повторного использования

**Критерии готовности спринта:**
- Кнопка "Загрузить файл" во вкладке "Анализ" работает
- Вкладка "Анализ" умеет работать как минимум с двумя типами источников
- Из вкладки "Загрузка фактов" в Анализ передается именно отдельный экспортируемый слой
- Пользователь видит, какой источник сейчас активен
- При ошибках загрузки и несовместимости источника показываются понятные сообщения
- После перезапуска приложения активный источник и его состояние восстанавливаются корректно
- Правая таблица Анализа одинаково работает для обоих типов источников

**В спринт сознательно не входят:**
- Реальное построение графиков
- Печать и экспорт графиков
- Комментарии к графикам
- Полный drag&drop
- Полная аналитическая фильтрация по всем типам данных
- Отрисовка нескольких разных таблиц напротив разных графиков

## Спринт 4. Построение и настройка графиков

**Цель:**
Перевести вкладку "Анализ" от подготовки конфигурации к реальной аналитической визуализации: пользователь должен иметь возможность настроить карточку и увидеть построенный график по подключенному источнику данных.

**Что уже имеется к началу спринта:**
- Рабочая вкладка "Анализ" с toolbar и сохранением состояния
- Поддержка двух типов источников данных:
	- отдельно загруженный файл анализа
	- экспортируемый слой из вкладки "Загрузка фактов"
- Табличное представление данных с поиском и сортировкой
- Карточки графиков с базовыми полями конфигурации
- Проверка совместимости карточек графиков со структурой текущего источника

**Backend-задачи:**
- Реализовать серверную подготовку данных для построения графиков по конфигурации карточки
- Добавить слой агрегации данных для графиков:
	- count
	- sum
	- avg
	- при возможности min и max без раздувания спринта
- Добавить валидацию конфигурации графика на сервере:
	- наличие обязательных полей
	- корректность выбранных колонок
	- совместимость агрегации с типом данных
- Подготовить API для получения данных конкретного графика на основе:
	- типа графика
	- источника данных
	- X
	- Y
	- group
	- agg
- Обеспечить безопасную обработку ошибок построения без падения вкладки "Анализ"

**Frontend-задачи:**
- Добавить реальную область визуализации для каждой карточки графика
- Реализовать построение как минимум базового набора графиков:
	- столбчатая диаграмма
	- линейный график
	- круговая диаграмма или scatter в зависимости от того, что проще и надежнее ложится на текущую структуру данных
- Связать поля карточки с реальным построением графика:
	- X
	- Y
	- group
	- agg
	- color
	- legend
	- labels
- Добавить локальные состояния графика:
	- график не настроен
	- недостаточно параметров
	- нет данных для построения
	- ошибка построения
	- график построен успешно
- Обеспечить корректную перестройку графика при изменении параметров карточки
- Сохранить текущую компактную компоновку интерфейса, чтобы список карточек и таблица не деградировали визуально

**Работа с логикой построения:**
- Визуализация должна строиться только по активному источнику Анализа
- Если источник изменился и карточка стала несовместимой, вместо графика должно показываться понятное сообщение о необходимости перенастройки
- Построение графика не должно уничтожать сохраненную конфигурацию карточки при ошибке
- Для первой версии важно обеспечить предсказуемость и понятность результата, а не максимальное число типов диаграмм

**Совместимость с предыдущими спринтами:**
- Не сломать загрузку и переключение источников из Спринта 3
- Не сломать сохранение типов анализа и карточек из Спринта 2
- Сохранить поиск, сортировку и метаданные источника в правой части вкладки
- Сохранить поведение предупреждений о несовместимости конфигурации

**Изменения в БД:**
- Новых обязательных таблиц не требуется, если текущая модель карточек уже хранит все параметры графика
- При необходимости можно расширить хранение карточки графика:
	- режим отображения
	- дополнительные настройки подписей
	- визуальные параметры, которые действительно используются при построении

**Критерии готовности спринта:**
- Пользователь может выбрать источник, настроить карточку и увидеть реальный график
- Как минимум 2–3 типа графиков строятся на реальных данных
- Изменение параметров карточки приводит к корректной перестройке графика
- При некорректной или неполной конфигурации пользователь видит понятное состояние, а не сломанный интерфейс
- При смене источника график либо корректно перестраивается, либо показывает понятную причину, почему это невозможно
- Вкладка "Анализ" остается стабильной и не ломает функциональность Спринтов 2–3

**В спринт сознательно не входят:**
- Печать графиков
- Экспорт графиков в файл или изображение
- Полноэкранный режим графиков
- Комментарии к графикам
- Полный drag&drop карточек
- Сложная система фильтрации отдельно для каждого графика
- Генерация готовых аналитических отчетов

## Спринт 5. Печать и экспорт графиков

Сделать результаты вкладки "Анализ" пригодными для передачи наружу: пользователь должен иметь возможность сохранить или распечатать построенные графики и связанные с ними агрегированные данные.

**Что уже имеется к началу спринта:**
- Во вкладке "Анализ" уже строятся базовые графики
- Карточки графиков поддерживают конфигурацию, совместимость с источником и локальные состояния
- Поддерживаются два источника данных Анализа:
	- отдельно загруженный файл анализа
	- экспортируемый слой из вкладки "Загрузка фактов"
- Для ряда конфигураций уже доступна табличная агрегация как часть preview

**Backend-задачи:**
- Реализовать подготовку экспортируемого представления графика и связанной табличной агрегации
- Добавить API для:
	- экспорта активного графика
	- экспорта всех видимых графиков текущего типа анализа
	- экспорта агрегированной таблицы по текущей карточке
- Подготовить единый контракт экспортных данных:
	- название графика
	- тип графика
	- источник данных
	- дата и время формирования
	- набор агрегированных значений
- Обеспечить безопасную обработку ошибок экспорта и печати без падения вкладки "Анализ"

**Frontend-задачи:**
- Добавить в карточки графиков действия для:
	- сохранения графика

	- заголовок

	- дата формирования
	- подписи и легенда, если они заданы
- Добавить понятные состояния:
	- экспорт недоступен, потому что график не настроен
	- источник недоступен
	- нет данных для печати или сохранения
	- экспорт завершён успешно

**Работа с результатом Анализа:**
- Экспорт должен формироваться из того же подтвержденного preview/агрегированного результата, который пользователь видит на экране
- Пользователь не должен получать файл, отличающийся по данным от текущего отображаемого графика
- В первую очередь важна надежность и предсказуемость результата, а не богатство форматов

**Совместимость с предыдущими спринтами:**
- Не сломать построение графиков из Спринта 4
- Не сломать выбор и переключение источников из Спринта 3
- Не сломать сохранение типов анализа и карточек из Спринта 2
- Сохранить корректную работу поиска, сортировки и правой таблицы данных

**Изменения в БД:**

- При необходимости можно расширить хранение карточки графика служебными параметрами экспорта:

**Критерии готовности спринта:**

- Пользователь может распечатать текущий график в читаемом виде
- Пользователь может выгрузить агрегированную таблицу, если она доступна для карточки
- Экспорт и печать работают без нарушения текущей логики построения графиков
- Ошибки экспорта отображаются явно и не ломают вкладку "Анализ"

**В спринт сознательно не входят:**
- Полноценный конструктор аналитических отчетов
- Многостраничные документы со сложной версткой
- Комментарии к графикам
- Совместная работа нескольких пользователей
- Расширенная система фильтров для каждого графика

## Спринт 6. Комментарии к графикам

**Цель:**
Добавить к графикам слой пользовательской интерпретации: пользователь должен иметь возможность сохранять краткие комментарии и пояснения к каждой карточке графика, чтобы результат Анализа содержал не только визуализацию, но и смысловой вывод.

**Что уже имеется к началу спринта:**
- Во вкладке "Анализ" уже строятся рабочие графики
- Для карточек доступны экспорт и печать результатов
- Типы анализа и карточки графиков сохраняются между сессиями

	- основной текст комментария

**Frontend-задачи:**

	- короткий заголовок
	- текст пояснения

	- комментарий заполнен
	- комментарий отсутствует
- Дать пользователю возможность редактировать, очищать и повторно сохранять комментарий без влияния на параметры построения графика
- Включить комментарий в HTML-экспорт и печатное представление графика

**Работа с аналитическим смыслом:**
- Комментарий должен относиться именно к конкретной карточке графика, а не к типу анализа целиком

**Совместимость с предыдущими спринтами:**

**Изменения в БД:**
- Расширение таблицы карточек графиков новыми текстовыми полями комментария
- Без добавления новых обязательных сущностей, если комментарий хранится на уровне карточки графика

**Критерии готовности спринта:**
- Пользователь может добавить комментарий к каждому графику
- Комментарий сохраняется и восстанавливается после перезапуска приложения
- Комментарий не ломает построение графика и не влияет на агрегацию данных
- Комментарий попадает в HTML-экспорт и печатную форму
- При смене источника и несовместимости графика текст комментария не теряется автоматически

**В спринт сознательно не входят:**
- Совместное редактирование комментариев
- История изменений комментариев
- Богатое форматирование текста
- Комментарии на уровне отдельных точек графика
- Система обсуждений и согласования
